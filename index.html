Guia de Git e GitHub

O que é o Git?

O Git é um sistema de controle de versão distribuído. Ele permite que várias pessoas trabalhem no mesmo projeto de forma organizada, salvando versões do código e possibilitando o trabalho colaborativo.

O que é o GitHub?

O GitHub é uma plataforma online de hospedagem de código que utiliza o Git. Ele permite armazenar, gerenciar e colaborar em projetos de desenvolvimento de software com controle de versão.

Diferenças entre Git e GitHub

  • Git: É uma ferramenta instalada localmente para controle de versões.
  • GitHub: É uma plataforma online para hospedagem de repositórios Git.

Principais comandos Linux para usar no Git Bash

  • ls - Lista arquivos e pastas
  • cd - Navega entre diretórios
  • mkdir - Cria uma nova pasta
  • touch - Cria um novo arquivo
  • rm - Remove arquivos ou diretórios
  • clear - Limpa o terminal
  • pwd - Mostra o caminho do diretório atual

Principais comandos do README do repositório aula_git

  • git init - Inicializa um repositório Git
  • git add . - Adiciona todos os arquivos para o stage
  • git commit -m "mensagem" - Cria um commit com uma mensagem
  • git branch -M main - Renomeia a branch principal para main
  • git remote add origin URL - Conecta o repositório local ao GitHub
  • git push -u origin main - Envia o código para o GitHub

Como criar um repositório no GitHub

  1. Acesse o GitHub e faça login.
  2. Clique em New repository.
  3. Preencha o nome do repositório e clique em Create repository.

Como ligar um projeto local ao GitHub

  1. Abra o terminal na pasta do projeto.
  2. Execute git init.
  3. Adicione os arquivos com git add ..
  4. Crie um commit com git commit -m "inicial".
  5. Adicione o repositório remoto com git remote add origin URL.

Como gerar uma versão com Git

  • Utilize o comando git commit para registrar uma nova versão do projeto.
  • Use git tag v1.0 para marcar uma versão específica.

Como enviar da máquina para o GitHub

  1. Adicione as mudanças: git add .
  2. Crie o commit: git commit -m "mensagem"
  3. Envie com: git push origin main
